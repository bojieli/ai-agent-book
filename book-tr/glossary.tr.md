# İngilizce - Türkçe Terimler Sözlüğü {.unnumbered}

Türkçe çeviri projesinde, yapay zeka ve yazılım dünyasında kalıplaşmış teknik terimlerin zorlama Türkçe kelimelerle çevrilerek anlam kaybına yol açmasını önlemek amacıyla **orijinal İngilizce terimlerin korunması** ve metin içindeki akışa göre Türkçe karşılıklarının parantez içinde veya yan yana belirtilmesi prensibi benimsenmiştir. 

Aşağıdaki tablo, open-source maintainer'ların ve çeviri denetçilerinin (özellikle Türkçe bilmeyen koordinatörlerin) çevirinin doğruluğunu İngilizce tanımlar üzerinden kontrol edebilmesi amacıyla 4 kolonlu bir yapıda (İngilizce Tanım dahil edilerek) hazırlanmıştır:

| İngilizce Terim | Metin İçi Önerilen Kullanım | Türkçe Açıklama / Anlamı | English Description |
| --- | --- | --- | --- |
| Ablation study | Ablation study (Ablasyon çalışması) | Bir sistemin veya modelin her bir bileşeninin sistem başarısına katkısını anlamak için bu bileşenlerin sırayla devre dışı bırakılarak test edilmesi. | A scientific test where components of a system are systematically removed to evaluate their individual contribution to overall performance. |
| Action Space | Action Space (Eylem Alanı) | Bir Agent'ın yapabileceği veya çağırabileceği tüm araç ve eylemlerin kümesi. | The set of all possible tools, functions, or actions that an Agent can invoke. |
| Agent | Agent (veya metin akışına göre Agent / Temsilci) | Kendi kararlarını verebilen, plan yapabilen, dış araçları (tool) çağırabilen ve gözlem sonuçlarına göre döngüsel olarak kendini düzelten yapay zeka sistemi. | An autonomous AI system capable of planning, executing steps, calling external tools, and self-correcting based on observations. |
| Agent bootstrapping | Agent bootstrapping (Ajanın kendi kendini başlatması/kurması) | Bir Agent'ın kendi kodunu veya yeni bir Agent sürümünü otonom olarak sıfırdan kurup başlatabilmesi. | An Agent otonomously setting up and launching its own code or a new Agent instance. |
| Agent skills | Agent skills (Ajan becerileri) | Ajanın tüm yetenek tanımlarını tek seferde yüklemek yerine ihtiyaç duydukça ilgili beceri veya prompt parçalarını dinamik olarak context'e yüklemesi. | Dynamically loading specific prompt segments or tool definitions on demand instead of stuffing them all into the system prompt. |
| Agentic RAG | Agentic RAG | RAG sistemlerinde bilgi getirme sürecinin (ne zaman, neyi ve nasıl arayacağını) bir Agent tarafından otonom olarak yönetilmesi. | RAG systems where the information retrieval process is autonomously managed and directed by an Agent. |
| Agentic workflow | Agentic workflow (Ajan tabanlı iş akışı) | LLM'in tek adımlı bir çıktı üretmesi yerine, planlama, yürütme, değerlendirme ve düzeltme adımlarını içeren ajan tabanlı dinamik süreçler. | Dynamic and iterative execution flows (planning, execution, self-correction) managed by an LLM instead of fixed, linear single-step generation. |
| Alignment | Alignment (Hizalama) | Yapay zeka modelinin davranış ve çıktılarının insan değerleri, hedefleri ve güvenlik kuralları ile uyumlu hale getirilmesi süreci. | The process of training and tuning AI models to ensure their behavior and outputs match human values, intentions, and safety guidelines. |
| Alignment tax | Alignment tax (Hizalama bedeli) | Güvenlik veya hizalama eğitimi (SFT/RLHF) sonrasında modelin bazı genel yeteneklerinde veya yaratıcılığında görülen hafif düşüş. | A slight drop in a model's general capability or creativity that can occur as a side effect of safety or alignment training. |
| API key | API key (API anahtarı) | Bir yazılım uygulamasının veya Agent'ın harici bir servise bağlanırken kimlik doğrulaması yapmasını sağlayan gizli anahtar. | A secret key used to authenticate an application or Agent with external services or APIs. |
| Artifact | Artifact | Agent'ın görev yürütürken ürettiği ara veya nihai çıktılar (kod dosyaları, raporlar, SQL sorguları, dokümanlar vb.). | Intermediate or final outputs generated by an Agent during task execution (e.g., code files, reports, database schemas). |
| Assistant role | Assistant role (Asistan rolü) | API mesaj yapısında, yapay zeka modelinin kendi yanıtlarını içeren ve konuşma geçmişini temsil eden rol tanımlayıcısı. | The API message role identifier indicating replies generated by the language model. |
| AST (Abstract Syntax Tree) | AST (Soyut Sözdizim Ağacı) | Kaynak kodun programlama dilinin dilbilgisi kurallarına göre oluşturulmuş ağaç yapısındaki ağaçsal temsili. | A tree representation of source code structure based on programming language syntax. |
| Asynchronous execution | Async execution (Asenkron yürütme) | Bir işlemin tamamlanmasını beklemeden diğer işlemlerin devam edebilmesini sağlayan paralel/eşzamanlı olmayan çalışma biçimi. | A non-blocking execution pattern where tasks run concurrently without waiting for previous steps to finish. |
| Audit log | Audit log (Denetim günlüğü) | Güvenlik, hata ayıklama ve izleme amacıyla sistemdeki tüm kritik olayların ve eylemlerin kronolojik kaydı. | A chronological record of critical system events and actions, used for security, tracing, and debugging. |
| AutoGen | AutoGen | Çoklu ajan (multi-agent) sistemleri ve işbirlikçi iş akışları tasarlamak için geliştirilmiş popüler bir açık kaynaklı yazılım çerçevesi (framework). | A popular open-source framework for building multi-agent systems and cooperative workflows. |
| Autonomous Agent | Autonomous Agent (Otonom Ajan) | İnsan müdahalesi olmadan, verilen yüksek seviyeli hedeflere ulaşmak için kendi eylemlerini planlayan ve dinamik olarak yürüten sistem. | An Agent that can independently plan and execute actions to achieve high-level goals without human intervention. |
| Benchmark | Benchmark (Kıyaslama testi) | Model veya Agent sistemlerinin performansını standartlaştırılmış görevler üzerinde ölçmek ve karşılaştırmak için kullanılan test seti. | A standardized suite of tests and tasks used to measure and compare model or system performance. |
| Bi-encoder | Bi-encoder (Çift kodlayıcı) | RAG sistemlerinde sorgu ve dokümanları iki ayrı modelle bağımsız vektörlere dönüştürerek hızlı benzerlik araması yapılmasını sağlayan yapı. | A RAG architecture that processes queries and documents independently into vectors for extremely fast similarity searches. |
| BM25 retrieval | BM25 retrieval (BM25 araması) | Anahtar kelime sıklığına ve doküman uzunluğuna dayalı, geleneksel ve son derece etkili bir bilgi getirme (retrieval) algoritması. | A highly effective, traditional keyword-based document retrieval algorithm based on term frequency and document length. |
| Browser automation | Browser automation (Tarayıcı otomasyonu) | Agent'ın web sayfalarında gezinmek, tıklamak veya veri kazımak için Playwright/Puppeteer gibi araçları otonom kullanması. | Autonomous control of a web browser (e.g., clicking, typing, scrolling) by an Agent using tools like Playwright or Puppeteer. |
| Cache hit / miss | Cache hit / miss (Önbellek isabeti / ıskalaması) | Talep edilen verinin önbellekte bulunması (hit) veya bulunmayıp sıfırdan hesaplanmak/getirilmek zorunda kalınması (miss) durumu. | Whether requested data is found in cache (hit) or needs to be calculated/retrieved from scratch (miss). |
| Centralized scheduling | Centralized scheduling (Merkezi programlama/zamanlama) | Çoklu ajan gruplarında, bir sonraki konuşacak ajanın merkezi bir karar mekanizması tarafından belirlendiği koordinasyon yöntemi. | A multi-agent coordination pattern where a central manager determines which Agent speaks or acts next. |
| Chain of Thought (CoT) | Chain of Thought (CoT - Düşünce Zinciri) | Modelin nihai bir yanıta varmadan önce, karmaşık problemleri adım adım çözerek mantıksal akıl yürütmesini sağlayan yöntem. | A prompt reasoning technique where the model generates step-by-step logical thoughts before reaching a final answer. |
| Chunking | Chunking (Parçalama) | Büyük metinlerin RAG sistemlerinde işlenebilmesi için anlamlı ve yönetilebilir küçük parçalara bölünmesi süreci. | The process of splitting large texts into smaller, meaningful segments for embedding and RAG retrieval. |
| Coding Agent | Coding Agent (Kodlama Ajanı/Temsilcisi) | Yazılım geliştirme süreçlerinde kod yazma, hata ayıklama, test etme ve dosya sistemi üzerinde değişiklik yapma yeteneğine sahip Agent. | An Agent specialized in writing, debugging, testing, and modifying source code within a file system. |
| Computer Use | Computer Use (Bilgisayar Kullanımı) | Agent'ın ekran görüntülerini analiz ederek bir insan gibi işletim sistemini, fareyi ve klavyeyi otonom kullanabilmesi yeteneği. | An Agent's capability to analyze screen pixels and autonomously operate an operating system, mouse, and keyboard like a human user. |
| Context | Context (Bağlam) | Modelin her bir çıkarım anında gördüğü bilgilerin tümü; system prompt, konuşma geçmişi, araç tanımları ve araç sonuçları bu kapsama girer. | The complete list of information (system prompt, history, tool schemas, tool outputs) available to the model during a single inference call. |
| Context Compression | Context Compression (Bağlam Sıkıştırma) | Context penceresinin dolmasını önlemek için konuşma geçmişinin semantik olarak veya token düzeyinde daraltılması. | Strategies to compress or prune conversation history to fit within context limits or optimize inference costs. |
| Context Engineering | Context Engineering (Bağlam Mühendisliği) | Modelin önüne koyulacak tüm verinin (sistem talimatları, araç tanımları, geçmiş, RAG çıktıları) sistemli ve maliyet/etkin yönetimi. | Systematic management and organization of context contents (prompts, tools, history, RAG outputs) for optimal cost and accuracy. |
| Context Window | Context Window (Bağlam Penceresi) | Modelin tek seferde kabul edebileceği maksimum girdi (token) kapasitesi limiti. | The maximum token capacity limit that a model can process in a single inference call. |
| CrewAI | CrewAI | Rol tabanlı multi-agent (çoklu ajan) ekipleri kurmak ve bunları iş süreçlerine entegre etmek için tasarlanmış bir yazılım kütüphanesi. | A role-playing multi-agent framework designed to orchestrate collaborative agent teams. |
| Cross-encoder | Cross-encoder (Çapraz kodlayıcı) | Sorgu ve aday dokümanı tek bir modelde birlikte işleyerek çok daha hassas ama maliyetli bir benzerlik skorlaması yapan yapı. | A RAG architecture that processes the query and a candidate document together, providing highly accurate but slower similarity scoring. |
| Defense in depth | Defense in depth (Derinlemesine savunma) | Tek bir güvenlik önleminin yetmeyeceği varsayımıyla, birden fazla uzmanlaşmış katmanın bir arada kullanıldığı katmanlı güvenlik yaklaşımı. | A layered security approach combining multiple specialized safeguards rather than relying on a single one. |
| Dense retrieval | Dense retrieval (Yoğun arama) | Metinlerin anlamını vektör gömmeleri (embedding) kullanarak semantik düzeyde arama yöntemi. | A semantic document retrieval method using high-dimensional vector embeddings of text. |
| Distillation | Model distillation (Model damıtma) | Büyük ve güçlü bir modelin (teacher) yetenek ve bilgilerinin daha küçük, hızlı ve ucuz bir modele (student) aktarılması süreci. | The process of transferring knowledge and capabilities from a large model (teacher) to a smaller, faster model (student). |
| DPO (Direct Preference Optimization) | DPO (Doğrudan Tercih Optimizasyonu) | Pekiştirmeli öğrenmede karmaşık bir ödül modeli kurmadan, doğrudan insan tercih verileri üzerinden modeli hizalayan post-training yöntemi. | An alignment method that directly optimizes model weights from human preference data without a separate reward model. |
| Edge Agent | Edge Agent (Sınır/Cihaz Üstü Ajan) | Sunucu yerine doğrudan kullanıcının cep telefonu, bilgisayarı veya yerel cihazı üzerinde çalışan Agent. | An Agent that runs locally on a user's device (phone, laptop) rather than on centralized cloud servers. |
| Embedding | Embedding (Vektör gömme) | Kelime, cümle veya dökümanların anlamlarını temsil eden çok boyutlu sayısal vektörler. | High-dimensional numerical vectors representing the semantic meaning of text chunks or objects. |
| Evaluation | Evaluation (Değerlendirme) | Bir Agent'ın başarısını, doğruluğunu, güvenliğini ve verimliliğini bilimsel ve ölçülebilir metotlarla test etme süreci. | The systematic process of measuring and validating Agent accuracy, performance, safety, and efficiency. |
| External Learning | External Learning (Dışsal öğrenme) | Bilgileri modelin ağırlıklarına yazmak yerine (fine-tuning vb.), harici dosyalarda, veritabanlarında veya belleklerde saklayıp context üzerinden modele sunma yöntemi. | Retaining and retrieving knowledge outside model weights (in vector databases, files, memories) rather than fine-tuning parameters. |
| Few-shot Adaptation | Few-shot Adaptation (Az örnekli uyarlama) | Modele prompt içinde birkaç başarılı örnek göstererek yeni bir görev modeline hızla adapte olmasını sağlama tekniği. | Prompting technique where a few complete examples of a task are included to help the model learn the pattern on-the-fly. |
| Fine-tuning | Fine-tuning (İnce ayar) | Hazır bir temel modeli, belirli bir veri kümesi üzerinde eğiterek daha özel bir göreve uyarlama süreci. | The process of training a base model on specialized datasets to adapt its parameters for specific tasks or domains. |
| Freeform Tool Calling | Freeform Tool Calling (Serbest Format Araç Çağırma) | Modelin araca katı JSON yerine ham metin (kod, sorgu vb.) gönderebildiği API düzeyinde araç çağırma biçimi. | An API-level tool-calling mode where the model sends raw text to a tool instead of strict JSON. |
| Function calling | function calling (Fonksiyon çağırma) | Modelin harici bir program fonksiyonunu çalıştırmak amacıyla parametreleri yapılandırılmış veri formatında (genellikle JSON) hazırlaması mekanizması. | A mechanism where the model outputs structured parameters (usually JSON) to execute external program functions or tools. |
| GRPO (Group Relative Policy Optimization) | GRPO (Grup Göreli Politika Optimizasyonu) | DeepSeek R1 ile popülerleşen, her adımda değer modeli (value model) kullanmak yerine grup içi karşılaştırmayla çalışan pekiştirmeli öğrenme algoritması. | An efficient RL algorithm (used in DeepSeek R1) that optimizes policies using group relative comparisons instead of an independent value model. |
| Guardrails | Guardrails (Güvenlik bariyerleri) | Agent sisteminde tehlikeli, istenmeyen veya kural dışı davranışları tespit edip engelleyen koruyucu yazılım katmanları. | Protective software layers that intercept inputs or outputs to prevent unsafe, toxic, or unauthorized Agent behavior. |
| Hallucination | Hallucination (Halüsinasyon) | Modelin doğru olmayan, uydurma veya gerçek dışı bilgileri kendinden emin bir şekilde üretmesi durumu. | A phenomenon where an LLM generates confident but factually incorrect or ungrounded responses. |
| Harness | Harness (Harness - Model dışı altyapı katmanı) | Modelin etrafını saran; kısıtlamalar, doğrulamalar, düzeltmeler, hata kurtarma, günlük kaydı ve güvenlik filtrelerini yöneten dış sistem altyapısı. | The external infrastructure surrounding the model, managing constraints, verification, correction, logging, and recovery. |
| Harness Engineering | Harness Engineering (Harness Mühendisliği) | Modeli güvenilir bir üretim sistemine dönüştürmek için model dışı koruyucu altyapının ve kontrol döngülerinin tasarlanması. | The systematic design and implementation of safety and reliability wrappers around raw language models. |
| HITL (Human-in-the-Loop) | HITL (Sürece insan dahil etme) | Karar veya yürütme sürecinin kritik noktalarında insanın onay vermesini veya müdahale etmesini sağlayan kontrol mekanizması. | A system design pattern where a human supervisor must confirm or reject critical decisions made by an Agent. |
| In-Context Learning | In-Context Learning (Bağlam İçi Öğrenme) | Modelin ağırlıklarını değiştirmeden, yalnızca bağlam penceresine yerleştirilen talimatlar ve örnekler yardımıyla anlık olarak öğrenmesi. | The ability of a model to temporarily learn new rules, patterns, or behaviors from prompts/context without changing its weights. |
| Inference | Inference (Çıkarım) | Eğitilmiş yapay zeka modelinin yeni girdiler karşısında çıktı üretme süreci. | The execution phase where a trained model processes inputs and generates output predictions or text. |
| Jailbreak | Jailbreak (Güvenlik kilidini kırma) | Kullanıcının modele manipülatif sorular sorarak güvenlik filtrelerini ve davranış kurallarını devre dışı bırakması atağı. | An attack where a user bypasses safety alignments of an LLM through manipulative prompt engineering. |
| KV Cache | KV Cache (Anahtar-Değer Önbelleği) | Transformer modellerinde her token üretiminde geçmiş tokenların anahtar (K) ve değer (V) matrislerinin yeniden hesaplanmasını önlemek için kullanılan önbellek mekanizması. | A transformer optimization that caches Key and Value representations of previous tokens to prevent redundant computations in generation loops. |
| Linter | Linter (Kod statik analiz aracı) | Kodlama Agent'larının yazdığı kodları çalıştırmadan önce sözdizimi, stil ve potansiyel hatalar açısından denetleyen yazılım aracı. | A static code analysis tool used by Coding Agents to detect syntax or style issues before execution. |
| LLM-as-a-Judge | LLM-as-a-Judge (Değerlendirici olarak LLM) | Agent çıktılarının doğruluğunu veya kalitesini otomatik olarak puanlamak için güçlü bir LLM'in hakem olarak kullanılması yöntemi. | An automated evaluation technique that uses a powerful LLM to evaluate, grade, or score outputs of other models or Agents. |
| Loop Engineering | Loop Engineering (Döngü Mühendisliği) | Agent'ların sürekli ve otonom çalışması için öneri, doğrulama, hata düzeltme ve bitiş kararlarını yöneten döngüsel sistem tasarımı. | System design patterns that manage continuous, autonomous Agent loops including planning, execution, verification, and correction. |
| Manager-Executor | Manager-Executor (Yönetici-Yürütücü) | İş birliğinde bir ana Agent'ın (Planner/Manager) görevleri planlayıp alt Agent'lara (Executor) dağıttığı mimari şema. | A multi-agent pattern where a coordinator Agent (Manager) decomposes tasks and delegates them to specialized Agents (Executors). |
| MCP (Model Context Protocol) | MCP (Model Bağlam Protokolü) | Anthropic tarafından geliştirilen, Agent'lar ile veri kaynakları ve araçlar arasında standartlaştırılmış, açık kaynaklı iletişim protokolü. | An open-source, standardized communication protocol developed by Anthropic to connect Agents with data sources and tools. |
| Model as Agent | Model as Agent (Agent Olarak Model) | Gelişmiş modellerin tool calling kararlarını post-training yoluyla yerleşik/native yetenek olarak içselleştirdiği, orkestrasyonun istemciden modele/sunucuya kaydığı paradigma. | A paradigm where advanced models internalize tool-calling decisions as a native capability via post-training, shifting orchestration from client to model/server. |
| Multi-agent collaboration | Multi-agent collaboration (Çoklu ajan iş birliği) | Birden fazla Agent'ın rollerine göre bölünerek karmaşık bir işi ortaklaşa çözmesi. | A system architecture where multiple specialized Agents work together, sharing information to solve complex tasks. |
| Multi-agent debate | Multi-agent debate (Çoklu ajan tartışması) | Birden fazla Agent'ın bir problemi çözmek veya bir metni doğrulamak için karşılıklı fikir beyan ederek ortak akla ulaşmaya çalıştığı yapı. | A pattern where multiple Agents discuss and critique each other's ideas to reach a synthesized, more accurate decision. |
| Observation | Observation (Gözlem) | Agent'ın gerçekleştirdiği bir eylem veya araç çağrısı sonrasında dış çevreden veya araçtan aldığı yanıt/veri. | The data or environment response received by an Agent after executing an action or calling a tool. |
| Observability | Observability (Gözlemlenebilirlik) | Sistemlerin iç durumunun loglar, izler (trace), metrikler ve denetim kayıtları ile dışarıdan ne kadar izlenebilir olduğu. | The ability to monitor, trace, and audit the internal states, actions, and decision histories of an Agent system. |
| Optimistic locking | Optimistic locking (İyimser kilitleme) | Concurrency kontrolünde, çakışmaların nadir olacağını varsayarak veriyi kilitlemeden işlem yapan, kaydederken değişiklik kontrolü gerçekleştiren mekanizma. | A concurrency control mechanism that allows concurrent operations without locks, verifying data consistency only at save time. |
| Orchestrator | Orchestrator (Orkestratör) | Çok adımlı iş akışlarında veya multi-agent sistemlerinde kontrol ve veri akışını koordine eden merkezi yapı. | A system component or pattern that manages control flow, data routing, and task execution in multi-step or multi-agent workflows. |
| Peer-to-peer collaboration | Peer-to-peer collaboration (Eşler arası iş birliği) | Ajanların merkezi bir yönetici olmadan doğrudan birbirleriyle mesajlaşarak ve bilgi alışverişinde bulunarak yardımllaştığı iş birliği yapısı. | A decentralized multi-agent pattern where Agents communicate and coordinate directly with each other without a central manager. |
| Poka-yoke | Poka-yoke | Toyota Üretim Sistemi'nden gelen, hataların tasarım yoluyla fiziksel/yapısal olarak imkânsız kılınması felsefesi. | A design philosophy (from the Toyota Production System) that makes errors physically impossible through design. |
| Policy | Policy (Politika) | Pekiştirmeli öğrenmede (RL), Agent'ın mevcut gözleme (observation) bakarak hangi eylemi (action) seçeceğini belirleyen karar verme stratejisi. | The decision-making strategy in Reinforcement Learning that maps observations to actions. |
| Post-training | Post-training (Sonradan eğitim) | Temel modelin (pre-trained) hizalanması ve özelleştirilmesi için yapılan SFT, RLHF, DPO gibi tüm eğitim süreçleri. | Alignment and tuning steps (such as SFT, RLHF, or DPO) performed on a base pre-trained model. |
| Pre-training | Pre-training (Ön eğitim) | Modelin büyük veri kümeleri üzerinde dil yeteneği ve genel dünya bilgisini edinmek üzere eğitildiği ilk aşama. | The initial stage of training where a language model learns general language patterns and world knowledge from large-scale text corpora. |
| Prompt | Prompt | Modelin çıktı üretmesini tetikleyen, kullanıcı veya sistem tarafından yazılmış her türlü yazılı girdi. | Any written input provided to a language model to trigger a text response or tool-calling decision. |
| Prompt Cache | Prompt Cache (Prompt Önbelleği) | API servis sağlayıcıları düzeyinde, aynı sistem promptunu ve araç tanımlarını paylaşan isteklerde KV cache'in tekrardan hesaplanmadan fatura indirimiyle sunulması. | An API-level optimization that caches the KV cache of stable prompt prefixes across multiple requests to reduce latency and costs. |
| Prompt Engineering | Prompt Engineering (Prompt Mühendisliği) | Modelin en doğru ve verimli çıktıyı üretmesi için doğal dilde yazılan talimatların sistematik olarak tasarlanması süreci. | The practice of designing and refining system and user prompts to optimize LLM outputs and behavior. |
| Prompt injection | Prompt injection (Prompt enjeksiyonu) | Harici veya kötü niyetli verilerin sisteme dahil edilerek Agent'ın asıl talimatlarının sabote edilmesi ve model kontrolünün ele geçirilmesi atağı. | An exploit where untrusted or hostile external inputs hijack the prompt context and manipulate the Agent's behavior. |
| Prompt leak | Prompt leak (Prompt sızıntısı) | Kötü niyetli bir sorgu sonucunda Agent'ın gizli tutulması gereken system promptunu veya özel talimatlarını açığa çıkarması zafiyeti. | An exploit where an attacker forces an Agent to output its internal system prompt or private instructions. |
| Proposer-Reviewer | Proposer-Reviewer (Önerici-İnceleyici) | Bir Agent'ın çıktıyı ürettiği (proposer), diğerinin veya bir doğrulayıcının ise bunu denetleyip düzelttiği (reviewer) kalite döngüsü paradigması. | A design paradigm where one Agent proposes a solution and a separate verifier (Reviewer) evaluates, critiques, and refines it. |
| Pydantic schema | Pydantic schema (Pydantic şeması) | Python'da veri tiplerini ve kurallarını belirleyerek Agent'ların ürettiği araç parametrelerinin doğrulanmasını sağlayan veri yapısı tanımı. | A Python library used to define data structures and validate parameters returned during tool calls. |

| Reflection | Reflection (Kendini değerlendirme / Yansıtma) | Modelin kendi ürettiği çıktıyı mantıksal olarak inceleyip hatalarını otonom olarak tespit etmesi süreci. | An Agent's capacity to evaluate its own outputs or thoughts otonomously and identify potential flaws or improvements. |
| Reinforcement Learning (RL) | Reinforcement Learning (RL - Pekiştirmeli Öğrenme) | Agent'ın çevresiyle veya bir test ortamıyla etkileşime girerek aldığı ödül ve cezalar doğrultusunda en iyi kararları almayı öğrenmesi süreci. | A machine learning paradigm where an Agent learns optimal decision policies through trial-and-error interactions and rewards in an environment. |
| Reranking | Reranking (Yeniden Sıralama) | RAG sistemlerinde ilk aşamada hızlıca getirilen aday dökümanların daha güçlü modellerle (Cross-encoder vb.) hassas şekilde yeniden sıralanması. | An optimization step in RAG that uses a more powerful cross-encoder model to re-score and re-order the top retrieved document candidates. |

| Scratchpad | Scratchpad (Karalama Defteri) | Modelin çıkarım yaparken veya araç çağırırken ara adımlarını, planlarını ve düşüncelerini yazıp okuduğu geçici bağlam alanı. | A temporary context area (like thought blocks) where a model records its intermediate plans, thoughts, and reasoning steps. |
| SFT (Supervised Fine-Tuning) | SFT (Denetimli İnce Ayar) | Modelin insan tarafından hazırlanmış kaliteli soru-cevap veya görev-çözüm örnekleri üzerinde eğitilmesi. | Fine-tuning a pre-trained model on high-quality human-written instruction-response pairs. |

| System Prompt | System Prompt (Sistem Talimatı) | Geliştirici tarafından yazılan, Agent'ın kimliğini, yetkilerini, kısıtlamalarını ve çalışma kurallarını belirleyen sabit sistem talimatı. | The developer-written instruction set that defines the Agent's identity, permissions, and operational constraints. |
